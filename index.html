What's New

Billing and Invoice Management

Enhanced - Order Details Report

Enhancement-1: Billing Status Column

We have added a new Billing Status column to the Order Details Report (Reports > Order > Order Details). This column is positioned between the Billing Amount and Order Status columns. Additionally, you can now use Billing Status as a filter when generating the report.

Enhancement-2: Added Two New Columns

Two new columns, Order Bill To Contact Number (header-level) and Bill To Contact Number (line-level), have been added to provide immediate visibility into contact information that was previously unavailable. While the existing Bill To Customer column remains unchanged, these additions allow users to access essential contact details directly within the report output, eliminating the need for manual cross-referencing and streamlining the billing review process.

Enhancement-3: All Expired Line Status Filter

We have added a new "All Expired" option to the Line Status filter. By selecting Order Status = Active combined with Line Status = All Expired, users can now generate a report of all active orders that no longer have any lines actively billing. This captures orders in which every line item is in an Expired, Terminated, or Canceled status. To maintain consistency, these individual statuses will still be labeled exactly as they appear in the UI within the final report output.

Rent Credit Policy — Early Lease Termination Feature (Customer specific)

RecVue now supports a termination credit policy that automates rent credit calculations when a rented asset is returned mid-billing cycle. Credits are tiered based on how early in the cycle the demobilization occurs — applying 30-day, 15-day, or 0-day credit rules — ensuring customers are billed fairly for actual usage. All credit transactions are linked back to the original invoice lines, supporting compliance with revenue recognition and a clean audit trail.

New Account and Site Date Columns Added to Customer Details Report

To provide a more comprehensive view of customer lifecycles, we have added four new date columns to the Customer Details Report: Account Start Date, Account End Date, Site Start Date, and Site End Date.

Bill-To Contact Number Column Added to Billing Details & Tax Report

Enhancement: A new Bill-To Contact Number column is now available in the Billing Details & Tax Report, located next to the Customer Account Name. This field automatically pulls the contact number from the Bill-To customer record associated with the contract. If no contact number is on file, the field will remain blank.

Business Value: This update streamlines reporting by allowing users to identify Bill-To contact information directly within the report, eliminating the need to cross-reference other records.

Provide Access to Billing Tier Details from Contract and Workbench UIs

Enhancement: We have streamlined how you view and manage billing tiers. You can now access Billing Tier details directly from the Order Line and Usage Workbench without needing to perform separate searches or navigate away from your current task.

Key Capabilities:

  • Direct Hyperlinks: The active Billing Tier name now appears as a clickable link on the Order Products page and the Usage Workbench.
  • Tier Overview Pop-up: Clicking the hyperlink opens a pop-up window displaying all tiers associated with that product, automatically sorted with the most recent effective dates at the top.
  • Real-Time Active Tier Display: The system automatically identifies and displays the currently active tier based on today's date compared to the tier's effective start and end dates.
  • Seamless Navigation: From the pop-up, you can click directly into the active tier to view or update tier details.

Cost Management

To ensure seamless operational flow, the RecVue Cost Management module is being developed to align directly with existing modules, creating an integrated ecosystem for financial data. The following are the enhancements of Cost Management.

Enhanced Cost Accruals for Estimated Return Services

We've enhanced RecVue's ability to internally calculate cost accruals for services related to estimated returns. This update removes the need for manual or external calculations by allowing the system to handle these accruals automatically based on your predefined rules.

Scope and Eligibility:

  • Included: Modular products on Lease contracts.
  • Excluded: Containers and sale-type contracts.

How it Works: The system now calculates two specific types of estimated final bill expenses and amortizes (spreads) them across the entire lease term:

  • Return Haulage Expense: Calculated at 100% of the delivery haulage expense for the same asset.
  • Other Dismantle Expenses: Calculated at 80% of the list price found in Order Management.

Key Accounting Details:

  • GL Allocation: Amortization is automatically directed to one of two GL expense accounts based on your specific "bill-how" rules.
  • Exclusions: To ensure accuracy, certain final bill item codes—such as cleaning fees—are excluded from these amortization rules.

This enhancement ensures your lease cost recognition is more accurate, automated, and aligned with the equipment's actual lifecycle.

Introduction of the Accrual Workbench for Consolidated Visibility

We are excited to introduce the Accrual Workbench, a centralized dashboard designed to streamline how you manage and view cost and revenue accruals.

Improvement: Previously, viewing accrual data and related accounting entries required navigating multiple screens or running separate reports. The new Accrual Workbench brings all this information together into a single, comprehensive view.

Key Features:

  • Centralized Access: Easily accessible via Menu > Accrual Workbench.
  • Unified Data: View both Accrual Schedules and their associated accounting entries side-by-side on one screen.
  • Detailed Reporting: The interface displays high-level workbench data as well as granular schedule details, following the standard Cost & Accrual column specifications.

This enhancement provides better visibility into your financial data, making it faster and easier to verify accrual accuracy and status.

Introduction of the Cost Workbench for Consolidated Visibility

We are excited to introduce the Cost Workbench, a centralized interface that gives you a clear, comprehensive view of your cost data in one place.

Improvement: Previously, tracking cost schedules and their related accounting entries required navigation through various parts of the system. The new Cost Workbench simplifies this by surfacing all relevant information on a single screen.

Key Features:

This enhancement saves time and improves accuracy by providing a "single source of truth" for monitoring your cost accounting data.

Accounting Visibility in the Cost & Accrual Workbench

We have added a dedicated set of accounting columns to the interface, allowing you to instantly review the financial side of cost and accrual records. What's Fixed

Contract Management

Audit History: Order Amendment Fix

Issue: Previously, when an order was amended or a status was changed via automated packages, the audit history would occasionally display incorrect or incomplete information.

Fix: The system now accurately captures and records every status change at both the order and line levels. Whether you update an order manually or through a process, the audit trail will now consistently show the correct details.

Fixed: "Sell To Site Does Not Have the Contact Specified" Error when updating Sell-To Location

Fixed a bug that prevented users from updating the Sell-To location on orders. Previously, attempting this change triggered the error: "Sell To Site does not have the contact specified."

Issue: When changing the Sell-To Location on the Order Details screen, the system mistakenly looked for a Customer ID instead of a Site ID. Because of this data mismatch, the system couldn't validate the site correctly and blocked the update. This primarily impacted workflows for partially billed or paid orders.

Fix: The search logic has been corrected to reference the proper Site ID field. This ensures the system correctly identifies and validates the location you select from the dropdown.

Result:

  • Users can now update Sell-To and Bill-To sites without encountering validation errors.
  • The fix applies to all order statuses, including those that are partially billed or paid.

Fixed Copy Line Issue for Reverse-Terminated Lines

Users encountered an error message when trying to modify the line date while copying a reverse-terminated line.

The root cause was identified as a special character entered in the Bill to Contact field, which caused an error during the update process.

The issue is now resolved, and the copy line functionality works correctly.

Fixed: Termination Date Restrictions on Evergreen Orders

We resolved an issue that prevented users from selecting a valid termination date for Evergreen orders. The system was incorrectly capping the date picker, preventing users from choosing dates beyond a specific (and often outdated) limit.

Issue: When the "Extend Evergreen End Date" program runs, it updates the end dates for individual lines, but not the overall order-level end date. Previously, the system checked the order-level date to determine the maximum allowable termination date. This caused the system to "lock" the calendar too early, disabling any dates beyond the order's original end date.

Fix: We updated the date-selection logic to be smarter about how it identifies the maximum allowed date:

  • For Evergreen Orders: The system now prioritizes the furthest line-level end date. If that isn't available, it looks at the order-level evergreen date.
  • For Standard Orders: The logic continues to use the standard effective end date as usual.

Result: Users can now successfully select and apply the correct termination dates for Evergreen orders without being restricted by the order-level date.

R26.1.1

Monthly Schedule Period Date Shift for Start Dates 28–30

Impacted Modules: Billing (Billing Schedules) & Partner Compensation (Payment Schedules)

Problem: For monthly schedules where the cycle start date falls on the 28th, 29th, or 30th of a month, the system was permanently shifting all future period dates after the schedule passed through February. Once the schedule reached February — where the intended start date did not exist — Oracle's ADD_MONTHS function snapped to the last day of the month, and that last-day-of-month anchor persisted for every subsequent period. This caused all periods after February to follow end-of-month logic rather than resume the original day-of-month pattern.

Impact:

  • Affected billing and payment schedules with Billing Cycle = Line Start Date where the start date was the 28th, 29th, or 30th.
  • Impacted both leap year and non-leap year scenarios.
  • ERP [Downstream] effects on customer billing systems and internal calculations.

Fix: The system now correctly handles February as a point-in-time exception. The February period compresses to fit the shorter month, but all subsequent periods resume the original start-day pattern.

We introduced a profile option [February Schedule Sync] to control the month-end payment behavior:

  • Disabled (default): System uses existing payment schedule logic.
  • Enabled: System applies the new and required logic for supported 28th-of-the-month scenarios.

Revenue Management

Corrected Revenue Allocation During POB Changes

Issue: When a user triggers the Change POB functionality, the SSP values are being recalculated unnecessarily. This recalculation distorts the SSP values, even though they are not dependent on the POB.

Fix: The Fix ensures that SSP values remain unchanged when Change POB is triggered. Since SSP values do not depend on POB, they are no longer recalculated, preventing distortion.

Issue: A 1-cent variance is observed in the waterfall Report when the waterfall type is set to Contractual.

Fix: The logic for calculating PERIOD_SELL_AMT in revenue schedules has been enhanced to correctly populate the value as 0.01 when no SSP revenue amount is available, thereby preventing zero-value assignment and eliminating the 1-cent variance. For all other scenarios where SSP revenue exists, the existing calculation logic remains unchanged to ensure consistency.

Result: The Waterfall Report is now 100% accurate. Whether you are running reports for Contractual Revenue or Net Revenue, your totals will match perfectly with no variances.

Note: This fix is included in the 26.2 update. Please ensure your system is updated to this version to see the changes.
