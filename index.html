Overview

This release document provides a consolidated overview of new features, enhancements, and resolved issues across various RecVue product components. Each feature is accompanied by a concise description highlighting its business impact and functional value. For detailed guidance on specific configurations or implementation steps, please contact the RecVue Support Team.

Note: This document also lists features and solutions provided to certain customers in interim release 26.1.1.

The most significant updates in this release focus on key modules including RecVue Foundation, Contract Management, Billing Management, Partner Compensation, and Revenue Management.

What's New

Billing & Invoice Management

Apex Bill Run Now Supports Multiple Selection for Key Fields

You can now select multiple values across key fields when running an Apex Bill Run (formerly Turbo Bill Run), giving you greater flexibility and control over which orders get processed.

Previously, the Bill Run could only target a single order at a time. You can now use multi-select filtering to target specific groups of orders. This capability is controlled via profile configuration — activate the "Enable Bill Run Multi Selection" profile option to enable it.

Multi-Select Fields:

  • Order Type
  • Business Unit
  • Order Category
  • Line Types
  • Product Name
  • Order #
  • Territory

For customers managing multiple Business Units or Order Types where multiple Bill Runs were previously required, this enhancement significantly reduces operational effort with flexible controls, smart AND-logic filtering, and targeted invoicing.

Enhanced Order Details Report

Enhancement 1 — Billing Status Column: A new Billing Status column has been added to the Order Details Report (Reports > Order > Order Details), positioned between the Billing Amount and Order Status columns. You can also use Billing Status as a filter when generating the report.

Enhancement 2 — Bill-To Contact Number Columns: Two new columns — Order Bill To Contact Number (header-level) and Bill To Contact Number (line-level) — have been added to provide immediate visibility into contact information. These additions allow users to access essential contact details directly within the report output, eliminating manual cross-referencing.

Enhancement 3 — "All Expired" Line Status Filter: A new "All Expired" option has been added to the Line Status filter. By selecting Order Status = Active combined with Line Status = All Expired, users can generate a report of all active orders with no lines actively billing — capturing orders where every line is in Expired, Terminated, or Canceled status.

Rent Credit Policy — Early Lease Termination (Customer Specific)

RecVue now supports a Cancellation Credit Policy that automates rent credit calculations when a rented asset is returned mid-billing cycle. Credits are tiered based on how early in the billing cycle the demobilization occurs — applying 30-day, 15-day, or 0-day credit rules — ensuring customers are billed fairly for actual usage.

All credit transactions are linked back to the original invoice lines, supporting revenue recognition compliance and maintaining a clean audit trail. The policy is configured at the division level to support specific business needs.

New Account and Site Date Columns Added to Customer Details Report

To provide a more comprehensive view of customer lifecycles, four new date columns have been added to the Customer Details Report: Account Start Date, Account End Date, Site Start Date, and Site End Date. These have been integrated into the existing layout in a logical sequence for easy reading and contextual relevance.

Bill-To Contact Number Column Added to Billing Details & Tax Report

A new Bill-To Contact Number column is now available in the Billing Details & Tax Report, located next to the Customer Account Name. This field automatically pulls the contact number from the Bill-To customer record associated with the contract. If no contact number is on file, the field remains blank.

This update streamlines reporting by allowing users to identify Bill-To contact information directly within the report, eliminating the need to cross-reference other records.

New Asset Workbench for Centralized Asset Management

A new Asset Workbench has been introduced, providing a single centralized location for tracking and managing assets throughout their lifecycle. It surfaces critical asset-level data — including current location and full history — and offers the following key capabilities:

  • Streamlined Search: Quickly find assets using Simple Search (Asset #, Serial #, or Order #) or Advanced Search to filter by Site Number and Product.
  • Billing Details Tab: Full visibility into billing schedules and lines (both scheduled and billed), including invoice numbers, sorted by billing date (newest first).
  • Fulfillment Tab: A complete list of all fulfillment activities, organized in descending order by start date.
  • Integrated Tracking: View an asset's location history alongside its associated contract and order details for a complete 360-degree view.

Access to Billing Tier Details from Contract and Workbench UIs

You can now access Billing Tier details directly from the Order Line and Usage Workbench without navigating away from your current task.

  • Direct Hyperlinks: The active Billing Tier name now appears as a clickable link on the Order Products page and the Usage Workbench.
  • Tier Overview Pop-up: Clicking the hyperlink opens a pop-up showing all tiers associated with that product, sorted with the most recent effective dates at the top.
  • Real-Time Active Tier Display: The system automatically identifies and displays the currently active tier based on today's date.
  • Seamless Navigation: From the pop-up, click directly into the active tier to view or update specific details (Buckets page).

What's New

Cost Management

To ensure seamless operational flow, the RecVue Cost Management module is being developed to align directly with existing modules, creating an integrated ecosystem for financial data. The following are the enhancements of Cost Management.

Enhanced Cost Accruals for Estimated Return Services

RecVue now internally calculates cost accruals for services related to estimated returns, removing the need for manual or external calculations. The system handles these accruals automatically based on predefined rules.

Scope and Eligibility:

  • Included: Modular products on Lease contracts.
  • Excluded: Containers and sale-type contracts.

How It Works: The system calculates two types of estimated final bill expenses and amortizes them across the entire lease term:

  • Return Haulage Expense: Calculated at 100% of the delivery haulage expense for the same asset.
  • Other Dismantle Expenses: Calculated at 80% of the list price found in Order Management.

Key Accounting Details: GL Allocation is automatically directed to one of two GL expense accounts based on your specific "bill-how" rules. Certain final bill item codes — such as cleaning fees — are excluded from amortization. This enhancement ensures lease cost recognition is more accurate, automated, and aligned with the equipment's actual lifecycle.

Introduction of the Accrual Workbench for Consolidated Visibility

The new Accrual Workbench is a centralized dashboard designed to streamline how you manage and view cost and revenue accruals. Previously, viewing accrual data and related accounting entries required navigating multiple screens or running separate reports.

  • Centralized Access: Easily accessible via Menu > Accrual Workbench.
  • Unified Data: View both Accrual Schedules and their associated accounting entries side-by-side on one screen.
  • Detailed Reporting: The interface displays high-level workbench data as well as granular schedule details, following the standard Cost & Accrual column specifications.

Introduction of the Cost Workbench for Consolidated Visibility

The new Cost Workbench is a centralized interface that gives you a clear, comprehensive view of your cost data in one place. Previously, tracking cost schedules and their related accounting entries required navigating through various parts of the system.

  • Centralized Access: Found easily under Revenue > Cost Workbench.
  • Unified View: See Cost Schedules and their associated Accounting entries side-by-side without switching screens.
  • Deep Visibility: The workbench includes both high-level summary columns and granular detail columns, allowing you to drill down into the specifics of each cost schedule.

This enhancement saves time and improves accuracy by providing a single source of truth for monitoring your cost accounting data.

Accounting Visibility in the Cost & Accrual Workbench

You can now view accounting entry details directly within the Cost & Accrual Workbench. A dedicated set of accounting columns has been added to the interface, allowing you to instantly review the financial side of cost and accrual records without navigating between different screens.

  • In-Context Accounting: Review GL-level details alongside accrual data on the same screen.
  • Streamlined Auditing: Eliminates the need to cross-reference separate accounting screens.
  • Single Location Management: Manage, audit, and verify cost and accrual records from one place.

RecVue Foundation

Automated User Management with SCIM 2.0

RecVue now supports SCIM 2.0 (System for Cross-domain Identity Management), allowing your organization to connect RecVue directly to your Identity Provider — such as Okta, Azure AD, OneLogin, or Google Workspace — to automate user lifecycle management.

  • Automated User Lifecycle Management: Users are automatically created, updated, or deactivated in RecVue the moment changes are made in your identity provider. This eliminates manual user setup and ensures access is removed instantly when a user leaves.
  • Secure OAuth 2.0 Authentication: All synchronization endpoints are protected by OAuth 2.0 tokens with strict access controls and a full audit log of every provisioning action.

This update streamlines IT operations, enhances security, and ensures that your user list is always accurate and up to date.

Improved Custom Objects Functionality

The Custom Objects framework has been significantly enhanced, giving you greater control, security, and flexibility to manage your unique data within the system.

Navigation Relocated:

New Navigation Old Navigation
Main menu > Extensions > Custom Object Main menu > System Administrator > Custom
  • Streamlined Import: The import functionality has been moved directly into the Custom Objects UI. You can now download templates and trigger imports without navigating to a separate screen.
  • Granular Permissions: A permission-based access control model has been introduced. When a new custom object is created, three default permissions are automatically generated: View, Create, and Delete.

These updates transform Custom Objects into a more robust and flexible tool for managing specialized business data.

RecVue Hub Platform — Usage Hub

User Permissions, KPIs, and UI Updates

The interface name has been updated from "Dashboard Usage Hub" to "Usage Hub".

New Permissions:

  • ViewUsageHub: Lets users access the Usage Hub menu and view details. They can navigate freely but cannot edit existing batches or records.
  • ManageUsageHub: Enables users to add batches, reprocess data, and import files. They can also edit error records and save updates.

KPIs Added to Usage Hub Summary Screen:

  • Total: Number of usage batches that have been processed.
  • Success: Number of usage batches where all records processed successfully.
  • Error: Number of usage batches where all records have errors.
  • Warning: Number of usage batches with at least one record in error status.

Batch Name Field in Upload File UI: An optional Batch Name field has been added. If provided, it uses your entered name along with the system date and timestamp. If skipped, the system generates a default batch name.

The SFTP Source field name has been renamed to SFTP File Extension.

Usage Hub — Manage Workspace Column Updates

The Manage Workspace now comes with the following columns pre-configured by default:

  • Total — Total records in the Batch
  • Success — Total records processed successfully
  • Error — Total records in error status

API Support for Usage Hub Batches

RecVue now supports API-submitted usage data, enabling the automatic creation of batches in the Usage Hub in addition to Usage Hub DML. Both CISCO and Coupa import sources are supported, with validation triggered automatically after import — no manual steps needed. Failed validation records remain in the Usage Hub for manual correction.

New Profile Option — "Use Usage DML API for Usage Hub":

  • N (default): Uses the API for standard Usage DML operations.

Customer Specific: V3 Review Accounting Report Enhancement

The V3 Review Accounting Report has been updated so that the Debit and Credit columns now only include transactions that have been successfully transferred to the General Ledger (GL Transfer Flag = "Y").

  • Improved Accuracy: Report totals now align strictly with data successfully transferred.

What's Fixed

Contract Management

Audit History: Order Amendment Fix

Previously, when an order was amended or a status was changed via automated packages, the audit history would occasionally display incorrect or incomplete information.

The Fix: The system now accurately captures and records every status change at both the order and line levels. Whether you update an order manually or through a package, the audit trail will consistently show the correct details.

Fixed: Order Type Display Issue

Resolved an issue where the Order Type would disappear from the screen immediately after being selected during order creation. While the system successfully saved the selection to the database, the UI failed to display it.

The Fix:

  • Field Synchronization: Corrected a naming mismatch between front-end and back-end systems.
  • Standardized Lookup: The system now uses a standardized API to fetch order types based on your specific organization.
  • Better Error Handling: The UI will now remain stable and informative instead of failing silently.

Fixed: "Sell To Site Does Not Have the Contact Specified" Error

Resolved a bug that prevented users from updating the Sell-To location on orders, which previously triggered the error "Sell To Site does not have the contact specified."

The Issue: When changing the Sell-To Location, the system mistakenly looked for a Customer ID instead of a Site ID, blocking the update. This primarily impacted partially billed or paid orders.

The Fix: The search logic has been corrected to reference the proper Site ID field. Users can now update Sell-To and Bill-To sites without validation errors. The fix applies to all order statuses, including those partially billed or paid.

Fixed: Copy Line Issue for Reverse-Terminated Lines

Users encountered an error when trying to modify the line date while copying a reverse-terminated line. The root cause was a special character in the Bill to Contact field. The issue is now resolved and the copy line functionality works correctly.

Fixed: Termination Date Restrictions on Evergreen Orders

Resolved an issue that prevented users from selecting a valid termination date for Evergreen orders. The system was incorrectly capping the date picker at the order-level end date instead of the furthest line-level end date.

The Fix:

  • For Evergreen Orders: The system now prioritizes the furthest line-level end date. If unavailable, it uses the order-level evergreen date.
  • For Standard Orders: The logic continues to use the standard effective end date.

Improved Sorting in the Amendment Workbench

Previously, the Products grid in the Amendment Workbench sorted items only by line number, causing products from different orders to be mixed together.

The Fix: The grid now follows a two-step sorting process:

  • Primary Sort: Items are grouped by Order ID (most recent orders at the top).
  • Secondary Sort: Within each order, items are organized by Line Number in ascending order.

Fixed: New Lines Now Included in Future-Dated Terminations

Previously, if a user added new service lines to an order after a future termination date had been set, those new lines were left out — causing amendments to remain stuck in "Partially Processed" status.

The Fix:

  • Complete Processing: When an order is terminated, all associated lines — including any added after the initial amendment — are automatically terminated on the same future date.
  • Proactive Alerts: A warning message now appears if you try to add a new service line to an order already scheduled for termination.

What's Fixed

Billing & Invoice Management

Resolved GBS Execution for Non-Manual Order Lines

The Issue: For orders with recurring line types at manual frequency, users were unable to run GBS for lines with billing frequencies like monthly or quarterly.

The Fix: The GBS option at the order header level is now enabled, allowing users to run GBS for non-manual order lines even when manual lines are present. GBS automatically skips manual order lines.

Fixed: Evergreen Stub Schedule Causing Billing Failures

The Issue: For evergreen billing lines, the last schedule in the sequence was generated as a STUB schedule (partial/short-period). If that stub got billed before the evergreen line was extended, the downstream scheduling logic became corrupted.

The Fix: Changes were made to the evergreen end-date calculation logic — specifically how it handles the end date in relation to the Bill Offset setting. The last schedule in the sequence is now always generated as a standard schedule, preventing billing and scheduling failures upon line extension.

R26.1.1

Date Alignment Fix: Contract Schedules

Resolved a synchronization issue between contract dates and their corresponding billing schedules. Previously, if a user updated the Line Start Date or End Date after manual schedules had already been generated, the schedules would not automatically align with the new contract term.

The Fix: A new validation check has been introduced. If you attempt to change line dates, the system will now display a guidance message to ensure your contract duration and financial schedules always remain in sync.

What's Fixed

Partner Compensation

Bug Fix: Payment Schedules Not Updating on Vendor/Line End Date Extensions

Fixed a bug in the GPS system that prevented payment schedules from regenerating during the New Amendment process. Previously, if a contract was extended beyond five years or past its "Evergreen" date, the system would fail to create updated schedules.

This issue specifically impacted orders with Order Intent = BOTH and Pricing Type = Usage with Delivery Pricing lines. Payment schedules will now regenerate correctly through the new amended end date, regardless of how far into the future the contract is extended — for both partially paid and fully paid lines.

Resolved: Zero-Quantity Deliveries Now Reflected in Payment Schedules

Fixed a bug where importing a delivery with a quantity of zero for a previously paid line failed to update the payment schedules. Although the import would succeed technically, the system did not create the necessary "zero-quantity" adjustment entries.

The Fix: Zero-quantity deliveries are now correctly recorded as adjustment entries, ensuring payment schedules accurately reflect these imports and audit trails remain complete.

Resolved: Null Amounts Now Correctly Display as Zero

Fixed a display issue where payment amounts appeared blank (null) instead of showing 0 for deliveries with zero quantity or cost, regardless of whether data was brought in via Standard Delivery Import, DML, or the Usage Hub.

The system now correctly displays 0 in the payment schedule amount field for these cases, ensuring data consistency and a uniform experience across all delivery import methods.

Resolved: Incorrect Year Displayed in Amendment Pop-up

Fixed a visual bug where the Effective Date in the Amendment Update pop-up was incorrectly adding one year to the date shown. For example, a date saved as December 31, 2025 would appear as December 31, 2026. The database value was always correct, but the display was misleading. The pop-up now correctly displays the actual date stored in the system.

R26.1.1

Monthly Schedule Period Date Shift for Start Dates 28–30

Impacted Modules: Billing (Billing Schedules) & Partner Compensation (Payment Schedules)

The Problem: For monthly schedules where the cycle start date falls on the 28th, 29th, or 30th of a month, the system was permanently shifting all future period dates after passing through February. Once the schedule reached February — where the intended start date didn't exist — Oracle's ADD_MONTHS function snapped to the last day of the month, and that anchor persisted for every subsequent period.

The Fix: The system now correctly handles February as a point-in-time exception. The February period compresses to fit the shorter month, but all subsequent periods resume the original start-day pattern.

A new profile option [February Schedule Sync] has been introduced:

  • Disabled (default): System uses existing payment schedule logic.
  • Enabled: System applies the new logic for supported 28th-of-the-month scenarios.
R26.1.1

Pay Run Accuracy for In-Advance Payments

Resolved an issue where entering a value in the Attribute_1V field caused the Pay Run engine to incorrectly select pay lines beyond the "Pay Through Date" for In-Advance payment types. The engine now correctly identifies and selects only eligible pay lines based on your input.

What's Fixed

RecVue Foundation

Enhanced User-Specific Employee GET API

The user-specific Get Employee API has been improved to retrieve employee details by employee number, making it easier and faster to retrieve the required records.

Import Source Validation Fix

Resolved a validation error that prevented users from creating new Import Sources. Previously, the system blocked creation if the name matched an existing one, even when the Source IDs differed.

The Result: You can now successfully create Import Sources with duplicate names, provided they have unique Source IDs, offering more flexibility in how you label your data sources.

Missing Data in Billing Lines API

Resolved an issue where the Bill Run Billing Lines API (GET /api/v2.0/billrun/{id}/billinglines) returned null records for certain bill runs.

The Issue: An internal database mapping error caused the system to look for records using an incorrect identifier. Because this identifier was sometimes empty, the API failed to retrieve the associated billing data.

The Fix: The data mapping has been corrected to use the proper unique ID. The API now reliably returns all billing line records as expected.

What's Fixed

Revenue Management

Corrected Revenue Allocation During POB Changes

Resolved an issue where changing a Performance Obligation (POB) on RSSP lines caused the system to miscalculate net revenue. Previously, the "Change POB" process incorrectly used the total selling price to update SSP values, failing to account for revenue already allocated within the Revenue Contract. This could cause allocated revenue to exceed the original contract value (e.g., a $1.26M contract inflated to $2.4M).

  • Accurate Revenue Totals: RSSP values are now correctly limited by the total revenue allocated in the contract.
  • Prevention of Over-allocation: Revenue cannot exceed the original contract value during a POB change.
  • Reliable Reporting: Prevents incorrect impacts on Contract Asset and Contract Liability (CACL) balances.

Fixed: Waterfall Report 1-Cent Accuracy Issue

Resolved a small but impactful 1-cent ($0.01) discrepancy in the period columns of the Waterfall Report. This only occurred when selecting the Contractual Revenue type; Net Revenue reports were unaffected.

Root Cause: A field called PERIOD_SELL_AMT was accidentally defaulting to $0.00 instead of the actual value when it couldn't find a Standalone Selling Price (SSP).

The Fix: The logic for the Period Sell Amount calculation has been updated. If the system doesn't find an SSP Revenue Amount, it now automatically pulls the correct revenue amount instead of defaulting to zero. The Waterfall Report is now 100% accurate for both Contractual Revenue and Net Revenue report types.

Note: This fix is included in the 26.2 update. Please ensure your system is updated to this version to see the changes.
